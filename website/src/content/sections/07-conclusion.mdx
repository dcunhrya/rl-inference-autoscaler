## Conclusion

We set out to learn **scaling policies** for ML inference that balance **replica cost** and **latency** under bursty traffic. The key ingredients were:

1. A **realistic simulator** with cold starts, queueing, and trace-driven RPS  
2. A clear **MDP reward** with tunable α, β, and γ  
3. **Distributed RL training** via Ray RLlib (PPO and DQN)  
4. Rigorous **benchmarks** against classical baselines  

On a held-out GPU-style trace, **PPO** dramatically outperformed fixed, HPA-style, and greedy policies in episode return—demonstrating that RL can learn temporal structure that myopic heuristics miss.

**Next steps (Phase 3):**

- Export trained policies from Ray checkpoints  
- Deploy inference behind **Ray Serve** with a FastAPI metrics endpoint  
- Shadow-mode comparison against production autoscalers
